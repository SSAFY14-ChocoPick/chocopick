# ChocoPick 요구사항 분석서 v5 (최종)

## 0. 문서 개요
- 목적: ChocoPick Android 앱 개발을 위한 기능/비기능 요구사항과 정책을 정의한다.
- 범위: 주문 생성 및 픽업(매장/포장) 흐름 중심, 결제/정산 제외
- 기술 연동: Android(Kotlin), Google Maps, Firebase(Auth, FCM), Server(Spring Boot), On-device AI(HuggingFace)

---

## 1. 사용자(Actor)
- 사용자(고객): 회원가입/로그인, 매장 탐색/검색/선택, 상품 조회/담기, 장바구니, 주문, 주문 상태 확인, 픽업 안내 확인, 리뷰 작성/수정/삭제, AI 챗봇/리뷰요약 사용
- 시스템(서버/FCM): 주문 저장, 주문 상태 변경 처리, 푸시 알림 발송, 리뷰 권한 검증(본인만 수정/삭제)
- (선택) 관리자/운영: 주문 상태 변경(임시 API/도구로 수행 가능)

---

## 2. 핵심 정책(확정 사항)

### 2.1 회원/로그인 정책
- 로그인 방식: 아이디/비밀번호
- 앱 재실행 후에도 로그인 유지
- 로그아웃 기능 제공

### 2.2 매장 탐색 정책
- 매장 화면은 지도 + 리스트를 동시에 제공한다.
- 리스트 기본 정렬은 거리순(가까운 순)이다.
- 상단 검색창으로 매장 이름 검색이 가능하다.
- 기본은 “내 주변 매장”을 노출한다(현재 위치 기반).

### 2.3 위치 권한 거부 정책
- 위치 권한이 거부되거나 위치 획득 실패 시,
  - 본사 좌표를 현재 위치처럼 사용하여 지도 중심을 설정한다.
  - 매장 리스트도 본사 기준으로 제공한다(본사를 최상단 포함 가능).
  - “위치 권한이 없어 본사 기준으로 표시 중” 안내를 표시한다.

### 2.4 장바구니/매장 변경 정책
- 장바구니는 단일 매장 기준으로 유지한다.
- 매장을 변경하면 장바구니를 비우고 새 매장 기준으로 다시 시작한다.

### 2.5 주문 취소 정책
- 주문 생성 이후 사용자는 주문 취소를 할 수 없다.
- 주문 상세/완료 화면에 “주문 후 취소 불가” 안내 문구를 표시한다.

### 2.6 주문 상태 정책
- 주문 상태는 4단계로 관리한다.
  - RECEIVED → PREPARING → READY → PICKED_UP
- 상태 변경은 서버가 단일 진실원천(SSOT)이며, 클라이언트는 조회만 한다.
- (권장) 테스트 모드에서 주문 생성 후 일정 시간 뒤 READY로 자동 전환 가능(서버/클라 중 택1)

### 2.7 알림 정책
- READY(픽업 준비 완료) 상태로 변경되면 푸시 알림을 발송한다.
- 알림 클릭 시 해당 주문 상세 화면으로 이동한다.

### 2.8 주문 방식 정책(중요)
- 주문 방식은 2가지:
  - STORE(매장 주문): NFC 테이블 태깅 필수, tableNo 주문마다 필수 저장
  - TOUT(포장 주문): NFC 필요 없음, 거리가 200m 이상이면 경고 후 주문 가능
- 거리 기준: 200m 고정

### 2.9 픽업 예정 시간 정책
- 주문 생성 시 pickupEtaAt은 기본 “현재 시각 + 30분”
- 테스트 모드에서는 “30초 후”로 설정 가능(플래그)

### 2.10 리뷰 정책
- 리뷰는 별점(1~5) + 텍스트로 작성한다.
- 리뷰 텍스트는 최대 200자
- 리뷰 수정/삭제는 작성자 본인만 가능

### 2.11 온디바이스 AI 정책
- HuggingFace에서 온디바이스 AI 모델을 다운로드하여 로컬 추론으로 동작한다.
- AI 기능은 2가지:
  1) 메인 FAB 챗봇
  2) 상품 리뷰 요약(최근 30개 리뷰 입력 기준)
- 모델 미다운로드 상태에서 AI 기능 진입 시 다운로드를 유도한다.
- (권장) 리뷰 요약 결과는 캐싱하여 반복 요청 시 빠르게 제공한다.

## 3. 데이터 요구사항(초안)

### 3.1 Store
- storeId: String
- name: String
- address: String
- lat: Double
- lng: Double
- phone: String(선택)
- isHq: Boolean

### 3.2 Product
- productId: String
- storeId: String
- name: String
- imageUrl: String
- price: Int
- description: String(선택)

### 3.3 CartItem
- productId: String
- name: String
- price: Int
- qty: Int

### 3.4 Order
- orderId: String
- uid: String
- storeId: String
- orderType: String (STORE | TOUT)
- tableNo: Int? (STORE일 때 필수, TOUT일 때 null)
- items: List<CartItem>
- status: String (RECEIVED / PREPARING / READY / PICKED_UP)
- createdAt: Long
- pickupEtaAt: Long
- distanceMeter: Int(선택, 주문 당시 거리 기록)
- warningConfirmed: Boolean(선택, TOUT 원거리 경고 확인 여부)

### 3.5 Review
- reviewId: String
- productId: String
- uid: String
- rating: Int (1~5)
- content: String (최대 200자)
- createdAt: Long
- updatedAt: Long(선택)

### 3.6 ReviewSummary(선택, 캐시)
- productId: String
- summaryText: String
- summarizedCount: Int (최대 30)
- updatedAt: Long

## 4. 기능 요구사항 (FR)
작성 원칙: 설명 / 사전조건 / 처리 / 화면/출력 / 예외 / 수용 기준(AC)

---

### F01. 상품 목록 출력
- 설명: 선택한 매장의 상품 목록을 출력하여 사용자가 주문할 수 있다.
- AC:
  - 상품 리스트(이름/이미지/가격)가 표시된다.
  - 상품 클릭 시 상세 화면으로 이동한다.

---

### F02. 회원 가입(회원 정보 추가)
- 설명: 사용자는 아이디/비밀번호 기반으로 회원 가입할 수 있다.
- AC:
  - 필수 입력값 누락 시 가입이 불가하다.
  - 가입 성공 시 로그인 화면 또는 홈으로 이동한다.

---

### F03. 회원 아이디 중복 확인
- 설명: 가입 시 아이디 중복 여부를 확인할 수 있다.
- AC:
  - 이미 존재하는 아이디는 사용 불가 안내가 표시된다.
  - 사용 가능한 아이디는 사용 가능 안내가 표시된다.

---

### F04. 회원 로그인(세션 유지)
- 설명: 가입된 아이디/비밀번호로 로그인할 수 있고, 앱 재실행 후에도 로그인은 유지된다.
- AC:
  - 로그인 성공 시 홈으로 이동한다.
  - 앱 재실행 후 자동 로그인 상태가 유지된다.

---

### F05. 회원 로그아웃
- 설명: 사용자는 로그아웃할 수 있다.
- AC:
  - 로그아웃 시 로그인 화면으로 이동한다.
  - 이후 보호된 기능 진입 시 로그인 요구된다.

---

### F06. 상품 주문(주문 생성)
- 설명: 로그인한 사용자는 장바구니에 담긴 상품을 주문할 수 있다.
- 처리:
  - 주문 생성 시 orderId 발급
  - status는 RECEIVED로 저장
  - pickupEtaAt 저장(기본 +30분 / 테스트 모드 +30초)
- AC:
  - 주문이 성공하면 주문번호가 발급된다.
  - 주문 상태가 RECEIVED로 저장된다.
  - 픽업 예정 시간이 저장된다.

---

### F07. 회원 정보 조회(최근 주문 포함)
- 설명: 로그인한 사용자는 자신의 회원 정보를 조회할 수 있다.
- 최소 범위: 사용자 식별자(아이디/uid), 최근 주문 목록 섹션
- AC:
  - 로그인한 사용자만 조회 가능하다.
  - 최근 주문 내역이 표시된다.

---

### F08. 주문 상세 조회
- 설명: 주문 번호 기반으로 주문 상세(상품/수량/상태/매장/픽업예정시간)를 조회할 수 있다.
- AC:
  - 주문 상세 항목과 상태가 표시된다.

---

### F09. 상품별 정보 조회
- 설명: 상품별 이름/이미지/단가 등 상세 정보를 조회할 수 있다.
- AC:
  - 상품 상세 정보가 표시된다.

---

### F10. 리뷰 작성(별점 + 텍스트)
- 설명: 사용자는 상품에 리뷰를 작성할 수 있다.
- 입력:
  - rating: 1~5
  - content: 1~200자
- AC:
  - 로그인 사용자만 리뷰 작성이 가능하다.
  - 작성된 리뷰가 상품 상세에 표시된다.
  - 200자를 초과하면 작성이 불가하다(또는 입력 제한).

---

### F11. 리뷰 조회(상품별)
- 설명: 상품 상세에서 해당 상품의 모든 리뷰 목록을 조회할 수 있다.
- AC:
  - 리뷰 목록이 최신순(또는 정렬 기준 확정)으로 표시된다.

---

### F12. 리뷰 수정(본인만)
- 설명: 리뷰는 작성자 본인만 수정할 수 있다.
- AC:
  - 본인 리뷰에만 수정 UI가 노출된다.
  - 타인 리뷰 수정은 서버/클라 모두에서 불가하다.

---

### F13. 리뷰 삭제(본인만)
- 설명: 리뷰는 작성자 본인만 삭제할 수 있다.
- AC:
  - 본인 리뷰에만 삭제 UI가 노출된다.
  - 타인 리뷰 삭제는 서버/클라 모두에서 불가하다.

---

### F14. 장바구니 합계 표시
- 설명: 장바구니 하단에 총 수량 및 총 금액을 표시한다.
- AC:
  - 총 n개와 총액이 항상 정확하다.
  - 수량 변경/삭제 시 즉시 갱신된다.

---

### F15. 주문 취소 불가
- 설명: 주문 생성 이후 취소는 불가능하다.
- AC:
  - 취소 버튼/기능이 존재하지 않는다.
  - 주문 상세/완료 화면에 “주문 후 취소 불가” 안내가 표시된다.

---

### F16. 매장 화면(지도 + 리스트 + 검색)
- 설명: 사용자는 주변 매장을 지도와 리스트로 동시에 확인하고, 검색창으로 매장 이름 검색이 가능하다.
- AC:
  - 지도와 리스트가 동시에 표시된다.
  - 리스트는 거리순 정렬이다(권한 허용 시).
  - 검색 결과가 리스트에 반영된다.
  - 검색 결과가 없으면 안내가 표시된다.

---

### F17. 위치 권한 거부 처리(본사 기준)
- 설명: 위치 권한 거부 시 본사 좌표를 현재 위치처럼 사용하여 매장을 표시한다.
- AC:
  - 권한 거부 시에도 매장 화면이 정상 동작한다.
  - 본사 중심으로 지도/리스트가 표시된다.
  - 안내 문구가 표시된다.

---

### F18. 매장 선택 및 매장 변경 시 장바구니 초기화
- 설명: 사용자는 매장을 선택하여 주문을 진행하며, 매장 변경 시 장바구니는 비워진다.
- AC:
  - 선택한 매장이 주문 과정 전체에 유지된다.
  - 매장 변경 시 장바구니가 초기화된다.
  - 초기화 안내가 표시된다.

---

### F19. 주문 방식 선택(STORE / TOUT)
- 설명: 장바구니 화면에서 주문 방식을 선택할 수 있다.
- UI: 토글/세그먼트 버튼(매장 / T-out)
- AC:
  - 사용자가 주문 방식을 선택할 수 있다.
  - 선택한 값이 주문 데이터(orderType)에 저장된다.

---

### F20. STORE(매장) 주문: NFC 테이블 태깅 필수 + tableNo 필수 저장
- 설명: 주문 방식이 STORE일 때 주문하기 버튼을 누르면 NFC 태깅을 통해 테이블 번호를 받아야 주문이 가능하다.
- 처리 흐름:
  1) STORE 선택
  2) 주문하기 클릭
  3) “Table NFC를 찍어주세요.” 다이얼로그
  4) 태깅 성공 시 tag에서 `table:{번호}` 파싱
  5) tableNo를 주문 데이터에 저장 후 주문 생성
  6) 주문 성공 시 주문 상세로 이동
- 예외:
  - NFC 미지원 기기: 주문 불가 안내
  - 태깅 취소/실패: 주문 생성 불가
  - 태그 형식 오류: 오류 안내 후 재시도
- AC:
  - STORE 주문은 NFC 성공해야만 주문된다.
  - 주문마다 tableNo가 반드시 저장된다.
  - 주문 성공 시 주문 상세로 이동한다.

---

### F21. TOUT(포장) 주문: 거리 200m 이상이면 경고 후 주문 허용
- 설명: 주문 방식이 TOUT일 때, 사용자-매장 거리가 200m 이상이면 경고 팝업을 띄우고 확인 시 주문을 진행한다.
- 기준: 200m 고정
- 처리:
  - 거리 < 200m: 바로 주문 생성
  - 거리 ≥ 200m:
    - 팝업: “현재 고객님의 위치가 매장과 200m 이상 떨어져 있습니다. 정말 주문하시겠습니까?”
    - 버튼: 취소 / 확인
    - 확인 시 주문 생성, 취소 시 중단
- 예외:
  - 위치 권한 거부 시: 본사 좌표를 현재 위치처럼 사용하여 거리 계산(안내 문구 권장)
- AC:
  - 200m 이상이면 경고 팝업이 뜬다.
  - 확인 시 주문이 생성된다.
  - 취소 시 주문이 생성되지 않는다.

---

### F22. Beacon 기반 이용 알림(심화)
- 설명: 매장 내(5m 이내)에서 앱 사용 시 이용 알림 팝업을 표시한다.
- 정책:
  - 24시간에 1회만 표시한다.
- AC:
  - 5m 이내 조건에서 팝업이 표시된다.
  - 24시간 내 재노출되지 않는다.

---

### F23. 푸시 알림(FCM) 및 클릭 이동
- 설명: 주문 상태가 READY로 변경되면 푸시 알림을 수신하며, 알림 클릭 시 주문 상세로 이동한다.
- AC:
  - READY 전환 시 알림이 수신된다.
  - 알림 클릭 시 해당 주문 상세로 이동한다.

### F25. 온디바이스 AI 모델 다운로드/관리

- 처리:

  - 다운로드 진행률 표시
  - 다운로드 실패 시 재시도
- 예외:
  - 저장 공간 부족/네트워크 실패 시 안내
- AC:
  - 모델 다운로드/완료 상태를 사용자가 확인할 수 있다.
  - 모델이 없으면 AI 기능 진입 시 다운로드를 유도한다.
  - 다운로드 실패 시 재시도할 수 있다.

### F27. AI 리뷰 요약(최근 30개)
- 설명: 특정 상품의 리뷰를 요약하여 상품 상세에 요약 영역으로 보여준다.
- 입력 범위:
  - 최신 리뷰 기준 최대 30개를 입력으로 사용한다.
- 출력 포맷(고정):
  - 한줄 요약:
  - 장점 Top3:
  - 단점 Top3:
- 정책(권장):
  - 요약 결과 캐싱(동일 상품에 반복 요청 시 빠르게)
- 예외:
  - 리뷰가 0개면 요약 버튼/영역을 비활성 또는 “리뷰가 없습니다” 표시
  - 모델 미다운로드 상태면 다운로드 유도(F25 연동)
- AC:
  - 상품 상세에서 리뷰 요약 기능이 동작한다.
  - 요약은 지정된 포맷으로 출력된다.
  - 요약 입력은 최근 30개로 제한된다.

## 5. 비기능 요구사항 (NFR)

### 5.1 성능
- 매장 화면(지도+리스트) 로딩은 3초 내 목표(환경에 따라 변동)
- 리뷰 요약은 로딩 상태를 표시하고, 완료 시 결과를 표시한다(권장)

### 5.2 안정성
- 네트워크 실패 시 재시도 제공
- 예외 처리로 크래시 방지
- 다운로드/추론 실패 시 사용자 안내 제공

### 5.3 보안/권한
- 주문/리뷰는 uid 기반 접근 통제(서버 검증)
- 리뷰 수정/삭제는 작성자 본인만 가능(서버에서 최종 검증)
- 권한 거부 시 대체 UX 제공:
  - 위치 권한 거부 → 본사 기준 표시
  - 알림 권한(Android 13+) 거부 → 안내 및 설정 유도(권장)
  - NFC/Beacon 사용 시 필요한 권한/설정 안내(권장)

### 5.4 호환성
- Android 8.0(API 26)+ 권장
