# ChocoPick 요구사항 분석서

## 1. 목적
본 문서는 ChocoPick 애플리케이션 개발을 위한
기능 요구사항과 비기능 요구사항을 정의한다.

## 2. 사용자 정의
- 일반 사용자: 앱을 통해 상품을 주문하고 매장에서 픽업하는 사용자

## 3. 기능 요구사항

### F01. 로그인 / 로그아웃
Firebase Authentication을 이용한 로그인 기능을 제공한다.

완료 기준:
- 로그인 성공 시 홈 화면으로 이동한다.
- 앱 재실행 시 로그인 상태가 유지된다.
- 로그아웃 시 로그인 화면으로 이동한다.

---

### F02. 지도 기반 매장 조회
Google Maps를 이용하여 주변 매장을 지도에 표시한다.

완료 기준:
- 현재 위치 기준으로 매장 마커가 표시된다.
- 마커 선택 시 매장 상세 화면으로 이동한다.

---

### F03. 매장 선택
사용자는 하나의 매장을 선택하여 주문을 진행할 수 있다.

완료 기준:
- 선택한 매장이 주문 과정 전체에서 유지된다.
- 주문 정보에 매장 ID가 포함된다.

---

### F04. 상품 목록 조회
선택한 매장의 상품 목록을 조회할 수 있다.

완료 기준:
- 상품 이름과 이미지가 리스트로 표시된다.
- 상품 선택 시 상세 화면으로 이동한다.

---

### F05. 상품 상세 및 옵션 선택
상품의 상세 정보와 옵션을 선택할 수 있다.

완료 기준:
- 옵션 선택 후 장바구니에 추가할 수 있다.

---

### F06. 장바구니
선택한 상품을 장바구니에서 확인할 수 있다.

완료 기준:
- 상품 수량 변경 및 삭제가 가능하다.

---

### F07. 주문 생성
사용자는 장바구니의 상품을 주문할 수 있다.

완료 기준:
- 주문 생성 시 주문 번호가 발급된다.
- 주문 상태는 RECEIVED로 저장된다.

---

### F08. 주문 상태 조회
사용자는 자신의 주문 상태를 확인할 수 있다.

완료 기준:
- 주문 상태가 화면에 표시된다.

---

### F09. 푸시 알림
주문 상태 변경 시 푸시 알림을 제공한다.

완료 기준:
- 픽업 준비 완료 시 알림이 수신된다.

---

### F10. 픽업 안내
매장 위치와 픽업 정보를 제공한다.

완료 기준:
- 지도에서 매장 위치를 확인할 수 있다.
